Test that the server can start and respond to /health (per SERV-05).

The server requires these env vars to boot (even for /health, due to module-level imports):
- `DATABASE_URL` -- needed by `db/client.ts` which throws at import time if missing
- `BETTER_AUTH_SECRET` -- needed by `auth.ts` which throws at import time if missing
- `REDIS_URL` -- needed by BullMQ queue workers

**Step 1: Check for .env or running services**

```bash
# Check if PostgreSQL is available
pg_isready 2>/dev/null && echo "PG_AVAILABLE" || echo "PG_NOT_AVAILABLE"

# Check if Redis is available
redis-cli ping 2>/dev/null && echo "REDIS_AVAILABLE" || echo "REDIS_NOT_AVAILABLE"
```

**Step 2: Attempt boot test**

If PostgreSQL and Redis are available, create a temporary .env from .env.example and start the server:

```bash
cd robin
# Set minimal env vars for boot test
export DATABASE_URL="postgresql://localhost:5432/robin_dev"
export BETTER_AUTH_SECRET="test-secret-for-migration-boot-check"
export REDIS_URL="redis://localhost:6379"
export PORT=3099

# Start server in background, wait for it to be ready, then check /health
npx tsx src/index.ts &
SERVER_PID=$!
sleep 3

# Test health endpoint
curl -s http://localhost:3099/health
HEALTH_EXIT=$?

# Clean up
kill $SERVER_PID 2>/dev/null
wait $SERVER_PID 2>/dev/null

exit $HEALTH_EXIT
```

Expected /health response: `{"status":"ok","timestamp":"<ISO date string>"}`

**If PostgreSQL or Redis are NOT available:**
- Document that SERV-05 is verified structurally (TypeScript compiles, gateway facade in place, all modules present) but not functionally (boot requires running DB and Redis)
