Replace the entire contents of `robin/src/gateway/client.ts` with the gateway facade (per GATE-01, GATE-02). The original file has a top-level IIFE that throws `Error('GATEWAY_HMAC_SECRET env var is required')` at module import time -- this must be completely removed.

Write this exact content to `robin/src/gateway/client.ts`:

```typescript
import type { SearchResult } from '@robin/shared'
import { logger } from '../lib/logger.js'

const log = logger.child({ component: 'gateway' })

export const gatewayClient = {
  provision: async (_userId: string, _publicKey: string) => {
    log.debug('gateway facade: provision (no-op)')
    return { status: 'stub', userId: _userId }
  },

  write: async (_req: {
    userId: string
    path: string
    content: string
    message: string
    branch: string
    batch?: boolean
  }) => {
    log.debug('gateway facade: write (no-op)')
    return { path: _req.path, commitHash: 'stub', timestamp: new Date().toISOString() }
  },

  search: async (
    _userId: string,
    _query: string,
    _limit = 10,
    _minScore?: number,
    _repoPaths?: string[],
  ) => {
    log.debug('gateway facade: search (no-op)')
    return { results: [] as SearchResult[], count: 0 }
  },

  read: async (_userId: string, _path: string) => {
    log.debug('gateway facade: read (no-op)')
    return { path: _path, content: '', commitHash: 'stub' }
  },

  reindex: async (_userId: string) => {
    log.debug('gateway facade: reindex (no-op)')
    return { status: 'stub' }
  },

  batchWrite: async (_req: {
    userId: string
    files: Array<{ path: string; content: string }>
    message: string
    branch: string
  }) => {
    log.debug('gateway facade: batchWrite (no-op)')
    return {
      commitHash: 'stub',
      fileCount: _req.files.length,
      timestamp: new Date().toISOString(),
    }
  },
}
```

This facade:
- Exports `gatewayClient` with the same name and shape as the original (all 13 importing files work without changes)
- All 6 methods (provision, write, search, read, reindex, batchWrite) return resolved promises with structurally valid objects matching original return types
- No HMAC_SECRET validation, no HTTP calls, no `node:crypto` import
- Uses logger from `../lib/logger.js` for debug tracing
- `search` returns `{ results: [], count: 0 }` so routes return empty arrays
- `read` returns empty content so frontmatter parsers get defaults
- `write`/`batchWrite` return stub commit hashes (callers only log these)
